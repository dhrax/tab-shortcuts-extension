# Tab Shortcuts

Tab Shortcuts is a compact productivity panel to manage browser tabs in Brave/Chromium (Manifest V3).

## Description

Tab Shortcuts is a small extension that provides quick tab management actions from a popup: duplicate, mute/unmute, close groups of tabs, group/sort by domain, copy links, manage saved sessions and snippets, and optional automatic rules per domain (auto-mute, auto-pin, auto-group).

The extension is written in TypeScript and uses Manifest V3 service worker background scripts. It is designed to be lightweight and framework-free.

## Features

- Quick actions: duplicate tab, mute/unmute, mute others, pin/unpin, close duplicates/left/right/others
- Group and sort tabs by domain
- Tab search with multi-select bulk actions
- Save / export / import sessions and personal snippets
- Workspace management
- Copy links in multiple formats (Markdown, plain, HTML, JSON)
- Auto rules by domain (optional): auto-mute, auto-pin, auto-group
- Options page to configure defaults

## Installation (Brave / Chrome)

1. Build the extension:

```bash
npm ci
npm run build
```

2. Open Brave/Chrome and go to `chrome://extensions/` (or `brave://extensions/`).
3. Enable "Developer mode" (toggle in the top-right).
4. Click "Load unpacked" and select the `dist/` folder generated by the build.

Notes:
- Do not commit the `dist/` folder to your repository. It is generated by the build step.

## Build command

The project uses the following build command:

```bash
npm run build
```

This runs `tsc` and copies the `public/` assets into `dist/`.

## Release process (Git tags + GitHub Actions)

This repository contains GitHub Actions workflows that automate build and releases.

- CI (`.github/workflows/ci.yml`): runs on `push` and `pull_request` to `main`. It checks out the code, sets up Node.js 22, runs `npm ci` and `npm run build`, and uploads the `dist/` directory as a workflow artifact.
- Release (`.github/workflows/release.yml`): runs when you push a tag matching `v*` (for example `v1.0.0`). It builds the extension, zips `dist/`, creates a GitHub Release for the tag, and attaches the zip file as a release asset.

To create a release locally and push the tag:

```bash
# create an annotated tag
git tag -a v1.0.0 -m "Release v1.0.0"
# push the tag to trigger the release workflow
git push origin v1.0.0
```

GitHub Actions will build and publish the release artifact automatically.

## Keyboard shortcuts

The extension registers several commands in the manifest. Default shortcuts included in the manifest are shown below; users can reconfigure them in the browser's extension shortcuts UI.

| Action | Default shortcut |
|---|---:|
| Open popup action (`_execute_action`) | Alt+Shift+A |
| Duplicate current tab | Alt+Shift+D |
| Toggle mute current tab | Alt+Shift+M |
| Close other tabs | Alt+Shift+X |
| Mute other tabs | (unassigned) |
| Close duplicate tabs | (unassigned) |
| Close tabs to left | (unassigned) |
| Close tabs to right | (unassigned) |
| Group tabs by domain | (unassigned) |
| Sort tabs by domain | (unassigned) |
| Toggle pin current tab | (unassigned) |

To change shortcuts in Chrome/Brave: go to `chrome://extensions/shortcuts` (or the equivalent Brave page) and edit the shortcuts for this extension.
